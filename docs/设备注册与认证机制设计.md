# 设备注册与认证机制设计文档

## 概述

本文档描述了MateClaw桌面端的设备注册与认证机制，用于实现：
- 用户授权设备数量限制
- 设备心跳检测
- 异常设备识别
- 多设备管理

## 2. 设备ID生成策略

### 2.1 生成规则

设备ID格式：`DC-{HASH}-{TIMESTAMP}`

```
DC-A1B2C3D4-E5F6G7H8
```

| 部分 | 说明 | 示例 |
|------|------|------|
| DC | 前缀，标识Device Client | 固定值 |
| HASH | 设备特征哈希 | 8位十六进制 |
| TIMESTAMP | 生成时间戳 | 13位数字 |

### 2.2 设备特征采集

#### Web端特征
```typescript
const deviceInfo = {
  userAgent: navigator.userAgent,
  screen: `${screen.width}x${screen.height}x${screen.colorDepth}`,
  timezone: Intl.DateTimeFormat().resolvedOptions().timeZone,
  language: navigator.language,
  platform: navigator.platform
}
```

#### Electron端特征（预留）
```typescript
// 未来可扩展为：
const machineId = {
  cpuId: await getCpuId(),      // CPU序列号
  boardSn: await getBoardSn(),   // 主板序列号
  macAddress: await getMac()    // MAC地址
}
```

### 2.3 持久化策略

1. **首次生成**：使用本地存储 (`localStorage`)
2. **重启后读取**：从 `localStorage` 读取
3. **多浏览器隔离**：每个浏览器实例有独立设备ID
4. **用户切换**：同一设备ID，不同用户可绑定

## 3. 设备注册流程

### 3.1 流程图

```
应用启动
    │
    ▼
加载/生成设备ID ──────────────────┐
    │                              │
    ▼                              │
检查设备是否已注册                  │
    │                              │
    ├─ 是 → 检查用户登录状态        │
    │           │                  │
    │           ├─ 否 → 请先登录    │
    │           │           │      │
    │           └───────────┼──────┤
    │                       ▼      │
    │               调用 /device/authenticate
    │                       │      │
    │                       ▼      │
    │               检查设备数量限制
    │               │              │
    │               ├─ 超限 → 拒绝  │
    │               └─ 正常 → 成功  │
    │                       │      │
    └───────────────────────┼──────┘
                            ▼
                      启动心跳定时器
                            │
                            ▼
                       应用就绪
```

### 3.3 核心代码

```java
@Transactional
public DeviceAuthResponse registerDevice(DeviceRegisterRequest request, 
                                          Long userId, String username,
                                          HttpServletRequest httpRequest) {
    // 检查设备是否已存在
    Device existingDevice = deviceMapper.findByDeviceId(request.getDeviceId());
    
    if (existingDevice != null) {
        // 已注册，更新信息
        if (!existingDevice.getUserId().equals(userId)) {
            return fail("该设备已被其他用户绑定");
        }
        // 更新设备状态...
    } else {
        // 检查用户设备数量
        int currentCount = deviceMapper.countByUserId(userId);
        if (currentCount >= deviceLimit) {
            return fail("设备数量已达到上限");
        }
        // 创建新设备...
    }
    
    return success("设备注册成功", sessionToken);
}
```

## 4. 设备认证流程

### 4.1 认证时机

| 时机 | 说明 |
|------|------|
| 应用启动 | 检查设备是否已注册 |
| 用户登录 | 验证设备与用户绑定关系 |
| Token刷新 | 确保设备仍然有效 |

### 4.2 认证判断逻辑

```
设备认证请求
    │
    ▼
检查设备是否存在
    │
    ├─ 不存在 → 返回"设备未注册"
    │
    └─ 存在
          │
          ▼
      检查设备状态
          │
          ├─ banned → 返回"设备已被禁用"
          │
          └─ 正常
                │
                ▼
            更新心跳时间
                │
                ▼
            返回认证成功
```

## 5. 心跳机制

### 5.1 心跳策略

| 参数 | 值 | 说明 |
|------|------|------|
| 心跳间隔 | 5分钟 | 客户端定时发送 |
| 超时阈值 | 30分钟 | 服务端标记离线 |
| 检查频率 | 5分钟 | 定时任务扫描 |

### 5.2 心跳流程

```
┌──────────────┐
│   应用就绪    │
└──────┬───────┘
       │
       ▼ (等待5分钟)
┌──────────────┐
│   发送心跳    │ ──→ POST /api/v1/device/heartbeat
└──────┬───────┘
       │
       ▼
┌──────────────┐
│ 更新活跃时间  │ ──→ last_heartbeat = NOW()
└──────┬───────┘
       │
       ▼
   等待下次心跳 (循环)
```

### 5.3 离线检测

服务端定时任务：
```java
@Scheduled(fixedRate = 300000) // 每5分钟
public void markOfflineDevices() {
    LocalDateTime threshold = LocalDateTime.now().minusMinutes(30);
    List<Device> offlineDevices = deviceMapper.findOfflineDevices(threshold);
    
    for (Device device : offlineDevices) {
        device.setStatus("offline");
        deviceMapper.updateById(device);
    }
}
```


## 6. 设备数量限制

### 6.1 限制规则

| 用户类型 | 设备限制 | 说明 |
|----------|----------|------|
| 普通用户 | 5台 | 默认值 |
| 管理员 | 无限制 | 可配置 |

### 6.2 配置项

```yaml
# application.yml
mateclaw:
  device:
    limit: 5              # 设备数量限制
    heartbeat-timeout: 1800  # 心跳超时（秒）
```

### 6.3 超限处理

```
用户尝试注册第6台设备
         │
         ▼
    检查设备数量
         │
         ├─ 当前数量 < 限制 → 允许注册
         │
         └─ 当前数量 >= 限制
                │
                ▼
         返回错误信息
         "设备数量已达到上限(5台)，请先解绑其他设备"
                │
                ▼
           引导用户
           解绑旧设备
```

## 8. 数据库设计

### 8.1 表结构

```sql
CREATE TABLE mate_device (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    device_id       VARCHAR(64) NOT NULL UNIQUE COMMENT '设备唯一标识',
    device_name     VARCHAR(128) COMMENT '设备名称',
    device_info     VARCHAR(512) COMMENT '设备信息',
    device_type     VARCHAR(32) DEFAULT 'desktop' COMMENT '设备类型',
    user_id         BIGINT COMMENT '所属用户ID',
    username        VARCHAR(64) COMMENT '所属用户名',
    status          VARCHAR(16) DEFAULT 'online' COMMENT '状态',
    last_heartbeat  DATETIME COMMENT '最后活跃时间',
    registered_at   DATETIME COMMENT '注册时间',
    is_current      TINYINT(1) DEFAULT 0 COMMENT '是否当前设备',
    ip_address      VARCHAR(64) COMMENT 'IP地址',
    os_info         VARCHAR(128) COMMENT '操作系统信息',
    app_version     VARCHAR(32) COMMENT '应用版本',
    create_time     DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time     DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted         TINYINT(1) DEFAULT 0,
    
    INDEX idx_user_id (user_id),
    INDEX idx_status (status),
    INDEX idx_last_heartbeat (last_heartbeat)
);
```

### 8.2 状态说明

| 状态 | 说明 | 触发条件 |
|------|------|----------|
| online | 在线 | 正常注册/心跳 |
| offline | 离线 | 30分钟无心跳 |
| banned | 禁用 | 管理员禁用 |
| unbound | 已解绑 | 用户主动解绑 |

## 9. 安全考虑

### 9.1 设备ID安全

- **不可预测性**：使用哈希函数生成，无法反推设备信息
- **唯一性**：每个设备实例有唯一ID
- **持久性**：重启后ID不变

### 9.2 认证令牌

- **会话Token**：每次认证生成新的会话Token
- **有效期**：与用户登录Token同步
- **隔离性**：每个设备的Token独立

### 9.3 异常检测

| 场景 | 检测方式 | 处理措施 |
|------|----------|----------|
| 多用户同一设备 | 检查user_id | 拒绝绑定 |
| 设备数量超限 | count检查 | 拒绝注册 |
| 频繁注册 | 请求频率限制 | 限流处理 |
| 异常心跳 | 时间间隔检查 | 标记警告 |

## 10. 前端集成

### 10.1 设备Store初始化

```typescript
// main.ts 或 App.vue
import { useDeviceStore } from '@/stores/useDeviceStore'

const deviceStore = useDeviceStore()

// 应用启动时初始化
deviceStore.initialize()
```

### 10.2 登录流程集成

```typescript
async function handleLogin() {
  // 1. 先注册/认证设备
  await deviceStore.initialize()
  
  // 2. 然后进行用户登录
  const loginResult = await authApi.login(credentials)
  
  // 3. 更新设备绑定信息
  await deviceStore.authenticateDevice()
}
```

### 10.3 登出流程

```typescript
function handleLogout() {
  // 1. 停止心跳
  deviceStore.stopHeartbeat()
  
  // 2. 清除用户状态
  localStorage.removeItem('token')
  
  // 3. 保留设备状态，供下次登录使用
  // deviceStore.isRegistered 保持 true
}
```
