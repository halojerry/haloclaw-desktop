# MateClaw Desktop 自动更新部署指南

## 概述

本文档描述如何部署 MateClaw Desktop 的自动更新服务器。

## 架构

```
┌─────────────────┐     HTTPS      ┌─────────────────────┐
│  MateClaw App  │ ────────────▶  │  Update Server     │
│  (electron-     │   检查/下载    │  (Nginx/CDN)       │
│   updater)      │ ◀───────────  │                    │
└─────────────────┘   latest.yml  └─────────────────────┘
                                         │
                                         ▼
                                  ┌─────────────────┐
                                  │  版本文件存储   │
                                  │  ├── latest.yml │
                                  │  ├── win/       │
                                  │  │   └── *.exe   │
                                  │  ├── mac/       │
                                  │  │   └── *.dmg   │
                                  │  └── linux/     │
                                  │      └── *.AppImage
                                  └─────────────────┘
```

## 1. 服务器配置

### 方案一：Nginx 静态文件服务器

```nginx
server {
    listen 443 ssl http2;
    server_name updates.mate.vip;

    # SSL 证书配置
    ssl_certificate /etc/nginx/ssl/updates.mate.vip.crt;
    ssl_certificate_key /etc/nginx/ssl/updates.mate.vip.key;
    ssl_protocols TLSv1.2 TLSv1.3;

    # 更新文件目录
    root /var/www/mateclaw-updates;
    autoindex on;

    # CORS 头（可选）
    add_header Access-Control-Allow-Origin *;

    # 缓存配置
    location ~* \.(yml|exe|dmg|AppImage)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }

    # Gzip 压缩
    gzip on;
    gzip_types text/plain application/yaml;
}
```

### 方案二：使用 CDN（推荐）

推荐使用阿里云 OSS、AWS S3 或 Cloudflare：

```javascript
// S3 桶策略示例
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadGetObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::mateclaw-updates/*"
    }
  ]
}
```

## 4. 生成签名

### 计算 SHA512

```bash
# Linux/macOS
sha512sum MateClaw\ Desktop-1.0.419-Setup.exe

# Windows (PowerShell)
Get-FileHash -Algorithm SHA512 MateClaw Desktop-1.0.419-Setup.exe
```

## 5. 构建并发布更新

### 构建命令

```bash
# 构建新版本
npm version patch  # 或 npm version minor / npm version major
npm run dist

# 上传文件到更新服务器
scp release/*.exe user@updates.mate.vip:/var/www/mateclaw-updates/win/
scp release/*.dmg user@updates.mate.vip:/var/www/mateclaw-updates/mac/
```

### 更新 latest.yml

```bash
# 生成新的 latest.yml
cat > /var/www/mateclaw-updates/latest.yml << 'EOF'
version: 1.0.419
versionDate: '2026-04-20T10:00:00.000Z'
releaseDate: '2026-04-20'
releaseNotes: |
  ## 新功能
  - 新增自动更新功能

files:
  - url: win/MateClaw Desktop-1.0.419-Setup.exe
    sha512: <生成的SHA512哈希>
    size: <文件大小>

path: win/MateClaw Desktop-1.0.419-Setup.exe
sha512: <生成的SHA512哈希>
EOF
```

## 6. 自动部署脚本

```bash
#!/bin/bash
# deploy-update.sh

VERSION=$1
BUILD_DIR="./release"
UPDATE_SERVER="user@updates.mate.vip"
UPDATE_PATH="/var/www/mateclaw-updates"

if [ -z "$VERSION" ]; then
    echo "Usage: ./deploy-update.sh <version>"
    exit 1
fi

echo "Deploying version $VERSION..."

# 上传 Windows 安装包
echo "Uploading Windows installer..."
scp "$BUILD_DIR/MateClaw Desktop-$VERSION-Setup.exe" "$UPDATE_SERVER:$UPDATE_PATH/win/"

# 上传 macOS DMG
echo "Uploading macOS DMG..."
scp "$BUILD_DIR/MateClaw Desktop-$VERSION.dmg" "$UPDATE_SERVER:$UPDATE_PATH/mac/"

# 上传 Linux AppImage
echo "Uploading Linux AppImage..."
scp "$BUILD_DIR/MateClaw Desktop-$VERSION.AppImage" "$UPDATE_SERVER:$UPDATE_PATH/linux/"

# 生成并更新 latest.yml
ssh "$UPDATE_SERVER" "bash -s" << 'ENDSSH'
VERSION=$(basename "$UPDATE_PATH/win/"*.exe | sed 's/MateClaw Desktop-//;s/-Setup.exe//')
SHA512=$(sha512sum "$UPDATE_PATH/win/MateClaw Desktop-$VERSION-Setup.exe" | cut -d' ' -f1)
SIZE=$(stat -c%s "$UPDATE_PATH/win/MateClaw Desktop-$VERSION-Setup.exe")

cat > "$UPDATE_PATH/latest.yml" << EOF
version: $VERSION
versionDate: '$(date -u +%Y-%m-%dT%H:%M:%S.000Z)'
releaseDate: '$(date -u +%Y-%m-%d)'
releaseNotes: |
  ## 新版本 $VERSION
  
  - 功能优化
  - Bug 修复

files:
  - url: win/MateClaw Desktop-$VERSION-Setup.exe
    sha512: $SHA512
    size: $SIZE

path: win/MateClaw Desktop-$VERSION-Setup.exe
sha512: $SHA512
EOF
echo "latest.yml updated"
ENDSSH

echo "Deployment complete!"
```

### 生产验证

1. 确认 `latest.yml` 可访问：`curl https://updates.mate.vip/latest.yml`
2. 确认安装包可下载：`curl -I https://updates.mate.vip/win/MateClaw-Desktop-1.0.419-Setup.exe`
3. 检查日志确认更新流程正常工作

## 8. 故障排除

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 404 Not Found | 文件路径错误 | 检查 latest.yml 中的 URL |
| SHA512 不匹配 | 文件损坏或哈希错误 | 重新上传文件并更新哈希 |
| 更新检测失败 | CORS 限制 | 添加 CORS 头或使用代理 |
| 下载后无法安装 | 签名无效 | 确认使用正确的签名证书 |

### 日志位置

- **Windows**: `%APPDATA%\MateClaw Desktop\logs\`
- **macOS**: `~/Library/Logs/MateClaw Desktop/`
- **Linux**: `~/.config/MateClaw Desktop/logs/`

## 9. 安全建议

1. **HTTPS 必须**：始终使用 HTTPS 传输更新文件
2. **签名验证**：electron-updater 默认验证 SHA512 哈希
3. **防火墙**：仅允许 443 端口入站
4. **定期备份**：备份 latest.yml 和所有版本文件
5. **访问日志**：监控更新服务器的访问日志
